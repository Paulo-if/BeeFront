# Aula — HTML & CSS: Estrutura, Display e Centralização

---

## O que vamos aprender hoje?

Nessa aula vamos construir juntos uma página com foto de perfil centralizada. Parece simples, mas pelo caminho vamos entender como o navegador pensa, o que é display, como margin funciona, e por que centralizar uma imagem às vezes parece mais difícil do que deveria.

---

## 1. A estrutura do HTML

Antes de qualquer CSS, precisamos entender o esqueleto da nossa página.

```html
<!DOCTYPE html>
<html lang="pt-br">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Projeto01</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>

    <div id="container">
      <div id="profile">
        <img src="./assets/avatar.png" alt="">
      </div>
    </div>

  </body>
</html>
```

**O que cada parte faz:**

- `<!DOCTYPE html>` — avisa o navegador que esse é um documento HTML5.
- `<html lang="pt-br">` — a raiz da página. O atributo `lang` diz ao navegador (e ao Google) que o conteúdo é em português.
- `<head>` — área de configurações. O usuário não vê nada daqui diretamente.
- `<meta charset="UTF-8">` — garante que acentos e caracteres especiais apareçam corretamente.
- `<meta name="viewport" ...>` — essa linha é responsável pela responsividade. Sem ela, o celular renderiza a página como se fosse um desktop e tudo fica pequeno.
- `<link rel="stylesheet" href="style.css">` — conecta o CSS ao HTML. Mude o `href` para o nome do seu arquivo CSS se ele tiver outro nome.
- `<body>` — tudo que o usuário vê na tela fica aqui.

---

## 2. O que é uma `<div>`?

```html
<div id="container">
  <div id="profile">
    ...
  </div>
</div>
```

`div` é uma caixa genérica. O navegador não aplica nenhum estilo visual a ela por padrão — sem cor, sem borda, sem nada. Ela existe apenas para **agrupar e organizar** elementos, para que possamos estilizá-los via CSS.

Pense assim: a `div` é como uma caixa de papelão vazia. O CSS é o que pinta, redimensiona e posiciona essa caixa.

Usamos `id` para dar um nome único a cada div. Depois, no CSS, chamamos esse nome com `#`.

---

## 3. Conectando ao CSS — como funciona o seletor `#`

No CSS, quando você escreve:

```css
#container {
  width: 360px;
}
```

O `#` significa: *"encontre o elemento com esse id"*. Então `#container` seleciona a div que tem `id="container"` no HTML.

Já `#profile img` significa: *"encontre a tag `img` que está dentro do elemento com id `profile`"*. Isso se chama **seletor descendente**.

## 4. Background — colocando imagem de fundo

Existem duas formas de fazer a mesma coisa:

**Forma longa (mais legível no início):**
```css
body {
  background-image: url(./assets/bg-mobile.jpg);
  background-repeat: no-repeat;
  background-position: top center;
  background-size: cover;
}
```

**Forma curta (shorthand):**
```css
body {
  background: url(./assets/bg-mobile.jpg) no-repeat top center / cover;
}
```

As duas são equivalentes. A forma curta junta tudo em uma linha só. Com o tempo você vai preferir a curta, mas no começo a longa ajuda a entender o que cada propriedade faz.

- `no-repeat` — a imagem não se repete caso seja menor que a tela.
- `top center` — posiciona a imagem no topo e centralizada horizontalmente.
- `cover` — faz a imagem cobrir toda a área disponível, mesmo que precise ser cortada.

## 5. `width` e `margin: auto` — centralizando uma caixa

```css
#container {
  width: 360px;
  margin: auto;
}
```

Por padrão, uma `div` ocupa **100% da largura disponível** (display block). Quando damos um `width` menor, ela ocupa só aquele espaço — mas fica encostada à esquerda.

Para centralizar, usamos `margin: auto`. O que isso faz? Distribui o espaço vazio igualmente dos dois lados. É como dizer: *"coloque o mesmo espaço na esquerda e na direita"*.

> **Atenção:** `margin: auto` só centraliza horizontalmente. Para centralizar verticalmente, precisamos de outras técnicas (flexbox, grid).

A forma longa do mesmo código seria:
```css
margin-right: auto;
margin-left: auto;
```

`margin: auto` é apenas o shorthand das duas.

## 6. Display — a propriedade mais importante do CSS

Essa é a parte mais importante da aula. Cada elemento HTML tem um comportamento padrão de exibição, chamado **display**.

### `display: block`

Elementos com display block:
- Ocupam **toda a largura disponível** do pai.
- Empurram o próximo elemento para baixo (quebram linha).
- Aceitam `width`, `height`, e `margin` em todas as direções.

Exemplos que já são block por padrão: `div`, `p`, `h1`, `section`, `header`.

### `display: inline`

Elementos com display inline:
- Ocupam **só o espaço do seu conteúdo**.
- Ficam na mesma linha que outros elementos.
- **Não aceitam** `width` nem `height`. Margin vertical tem efeito limitado.

Exemplos que já são inline por padrão: `span`, `a`, `strong`, `img` (tecnicamente inline-block, mas se comporta como inline no fluxo).

### `display: none`

Remove o elemento da tela completamente. É como se ele não existisse para o navegador. Útil para mostrar/esconder elementos com JavaScript.

---

## 7. Por que `margin: auto` não centraliza a imagem diretamente?

```css
#profile img {
  width: 112px;
  /* margin: auto; não funciona aqui! */
}
```

A tag `<img>` tem display **inline** por padrão. Elementos inline não respeitam `margin: auto` para centralização horizontal da mesma forma que elementos block.

Para resolver isso, temos duas opções:

**Opção 1 — Mudar o display da imagem para block:**
```css
#profile img {
  display: block;
  width: 112px;
  margin: auto;
}
```

Agora a imagem se comporta como um bloco e `margin: auto` funciona.

**Opção 2 — Usar `text-align: center` no pai:**
```css
#profile {
  text-align: center;
}
```

`text-align` alinha o conteúdo inline de um elemento. Como a imagem é inline por padrão, ela fica centralizada. Mas **atenção**: se você mudar o display da imagem para block, essa abordagem para de funcionar — elementos block não são afetados por `text-align`.

---

## 8. `border-radius` — deixando a imagem redonda

```css
#profile img {
  border-radius: 50%;
}
```

`border-radius` arredonda os cantos de um elemento. Com `50%`, os quatro cantos ficam tão arredondados que o elemento vira um círculo (desde que largura e altura sejam iguais).

---

## 9. Resumo visual do fluxo

```
body (block — toda a largura da tela)
  └── #container (block — 360px, centralizado com margin: auto)
        └── #profile (block — ocupa os 360px do pai)
              └── img (inline por padrão → mudamos para block)
                    └── margin: auto → centraliza dentro do #profile
```

## 10. Código final comentado

**index.html**
```html
<!DOCTYPE html>
<html lang="pt-br">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Projeto01</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <div id="container">
      <div id="profile">
        <img src="./assets/avatar.png" alt="Foto de perfil">
      </div>
    </div>
  </body>
</html>
```

**style.css**
```css
body {
  /* Shorthand: imagem, repetição, posição / tamanho */
  background: url(./assets/bg-mobile.jpg) no-repeat top center / cover;
}

#container {
  width: 360px;    /* largura fixa */
  margin: auto;    /* centraliza horizontalmente na tela */
}

#profile img {
  display: block;      /* muda de inline para block */
  width: 112px;        /* define o tamanho da imagem */
  border-radius: 50%;  /* deixa redonda */
  margin: auto;        /* centraliza dentro do #profile */
}
```
